ketocoach-api

Fastcoach MVP API development workspace

Scripts

yarn build -- build the src into lib (i.e. prepare the functions to be deployed)
yarn serve -- build and serve the cloud functions locally
yarn shell -- build and serve the cloud functions locally and launch into an interactive shell
yarn start -- same as yarn shell
yarn deploy -- deploy the cloud functions to the firebase project (WARNING: This will overwrite the live cloud functions in the fastcoach-fc firebase environment! Please run this with care!)
yarn logs -- generate logs from the current cloud functions (generates local logs not real ones like you might see in the Stackdriver (GCP's logger for cloud functions))
yarn test -- run the current test suite using jest
build-docs -- build documentation (using typedoc) automatically into the docs folder
docs -- build documentation and automatically view it in a browser (using serve)

Notes

node version: This project expects to run using node 10 (10.18.1).
Make sure to run nvm use in the functions/ folder of this project to switch to node 10.18.1 automatically
(if you don't have it installed npm will raise an error, so run nvm install v10.18.1 in that case)

using nvm in npm scripts: It would be nice if we could have npm set the node version with nvm automatically in the scripts, however, (as far as I know) this is not possible because of how the script env works! (without setting your global default node version to match the project) ;-;
(As a workaround I've explicitly set the PATH to point at the npm config preset for the functions that run yarn to make them work in vscode)
